# 后续规划

更偏“为什么这么排、哪些更值钱”的分析，见 [`optimization-qa.md`](optimization-qa.md)。

## 1. 高优先级

### 1.1 电力知识库二期扩充

- 在当前第一版公开知识条目的基础上继续补充缺陷案例、SOP 和典型处置经验
- 将知识增强输出进一步沉淀为“异常报告 / 巡检摘要 / 缺陷上报草稿”
- 让项目更像“懂电力运维流程的 AI 助手”，而不只是会看指标

### 1.2 真实大模型接入

- 接入真实 LLM API
- 让知识增强回答在真实模型模式下跑通并稳定可控
- 保留本地规则分析作为安全兜底

### 1.3 Android 真机联调

- 安装 debug APK 到真机
- 验证局域网连接到共享网关
- 验证锁屏 / 切后台 / 弱网条件下的上报行为

### 1.4 客户端产品化补强

- PC 客户端补版本提示、本地日志入口和异常回放辅助信息
- 移动端补连接状态、本地说明和更适合展示的引导界面

### 1.5 闭环协作能力

- 将知识增强分析进一步接到 GitHub Projects / Issue / 运维报告草稿
- 让“异常识别 -> AI 建议 -> 留痕协作”链路更完整

## 2. 中优先级

### 2.1 页面答辩版优化

- 更统一的视觉语言
- 更适合演示的概览卡片
- 更突出的异常与高风险提示
- 更完整的设备详情页

### 2.2 文档与展示增强

- 增加系统流程图
- 增加答辩讲解词
- 增加样例截图
- 增加“系统设计与实现”章节素材

### 2.3 GitHub 工程化

- 配置 GitHub Issues 模板
- 配置 PR 模板
- 配置基础 CI 测试工作流
- 完善仓库首页导航

### 2.4 客户端产品化

- 个人 PC 客户端增加版本提示和本地日志入口
- 个人 PC 客户端继续增强自动重连与错误可见性
- 移动端增加更明确的连接状态与本地说明

### 2.5 backend manager 服务化包装

- 增加更直接的启动脚本 / 常驻运行包装
- 让页面外的后端托管方式更稳定、更容易交接
- 保持当前健康检查和状态文件能力继续可见

## 3. 低优先级

### 3.1 数据持久化升级

- 将 JSON 文件存储替换为 SQLite 或轻量数据库
- 支持跨会话历史曲线回放

### 3.2 真实接入扩展

- MQTT 替代 / 补充 HTTP 网关
- 阿里云 IoT 平台联调
- 更多终端设备模板

## 4. 建议近期执行顺序

1. 扩充电力知识库并生成更像业务文书的输出
2. 联通真实模型并完成稳定性验收
3. 完成 Android 真机联调与 PC 客户端小修
4. 补闭环协作能力和 GitHub 联动
5. 最后再补答辩版页面优化与材料整理
