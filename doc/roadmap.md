# 后续规划

更偏“为什么这么排、哪些更值钱”的分析，见 [`optimization-qa.md`](optimization-qa.md)。

## 1. 高优先级

### 1.1 真实大模型版聊天式 AI 助手

- 当前本地规则版聊天框已经完成
- 下一步要接入真实模型生成回答
- 保留当前本地工具层作为兜底能力
- 支持更自然的多轮追问与上下文记忆

### 1.2 真实大模型接入

- 接入真实 LLM API
- 让总结从本地模板升级为真实推理
- 保留本地规则分析作为安全兜底

### 1.3 MPC Skill 联调

- 将现有分析函数包装为标准 Skill 能力
- 支持模型通过 Skill 查询指定设备状态
- 支持技能返回结构化分析结果

### 1.4 Android 真机联调

- 安装 debug APK 到真机
- 验证局域网连接到共享网关
- 验证锁屏 / 切后台 / 弱网条件下的上报行为

### 1.5 聊天主流程 Skill 收敛

- 让聊天主流程尽量统一走 `dashboard_skill_adapter`
- 为后续真实 Skill / function calling 保持稳定入口
- 降低本地规则、Tool 层和适配层三处并行演化的复杂度

## 2. 中优先级

### 2.1 页面答辩版优化

- 更统一的视觉语言
- 更适合演示的概览卡片
- 更突出的异常与高风险提示
- 更完整的设备详情页

### 2.2 文档与展示增强

- 增加系统流程图
- 增加答辩讲解词
- 增加样例截图
- 增加“系统设计与实现”章节素材

### 2.3 GitHub 工程化

- 配置 GitHub Issues 模板
- 配置 PR 模板
- 配置基础 CI 测试工作流
- 完善仓库首页导航

### 2.4 客户端产品化

- 个人 PC 客户端增加托盘模式
- 个人 PC 客户端增加自动重连与错误提示
- 移动端增加更明确的连接状态与本地说明

### 2.5 backend manager 服务化包装

- 增加更直接的启动脚本 / 常驻运行包装
- 让页面外的后端托管方式更稳定、更容易交接
- 保持当前健康检查和状态文件能力继续可见

## 3. 低优先级

### 3.1 数据持久化升级

- 将 JSON 文件存储替换为 SQLite 或轻量数据库
- 支持跨会话历史曲线回放

### 3.2 真实接入扩展

- MQTT 替代 / 补充 HTTP 网关
- 阿里云 IoT 平台联调
- 更多终端设备模板

## 4. 建议近期执行顺序

1. 推送代码到 GitHub，先把展示面搭好
2. 联通真实模型并完成稳定性验收
3. 收敛聊天主流程到 Skill adapter
4. 完成 Android 真机联调与 PC 客户端小修
5. 最后再补答辩版页面优化与材料整理
