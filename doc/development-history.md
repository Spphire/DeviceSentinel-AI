## 关键演进路线

### 1. 从单场景演示到模板驱动平台

项目起初只是一个 SGCC 配电设备的本地演示版，后续逐步演进为模板驱动架构，使不同设备类型能够通过配置文件接入，而不需要每新增一种设备就重写页面逻辑。

### 2. 从单次随机值到连续时序监测

早期版本以单次随机生成数据为主，不适合展示设备“连续运行状态”。当前版本已经支持连续时序推进、故障窗口、离线窗口和多设备独立画像。

### 3. 从纯模拟到模拟与真实混合接入

在 SGCC 模拟设备之外，项目新增了真实设备接入通道，当前已经支持个人 PC、手机端和温湿度设备的真实遥测上报，并补上了桌面 GUI / Android APK 两条更适合展示的客户端交付形态。

### 4. 从硬编码页面到可配置化页面

设置弹窗、设备清单、模板选择、通讯方式、开发者模式等，已经逐步从代码中抽离成可配置项，并支持本地持久化恢复。

### 5. 从设备级 HTTP 配置到共享网关

真实设备最初按“每台设备单独填 host / port / path”建模，但这和实际共享网关架构不一致。当前版本已经收敛为单一共享 `/telemetry` 入口，由独立 backend manager 托管网关生命周期，设备本身只保留 `instance_id` 等自身信息。

## 当前阶段评价

目前项目已经具备一个较完整的“课程设计演示基线”：

- 有实时页面
- 有模拟数据
- 有真实设备
- 有异常分析
- 有 AI 风格总结
- 有文档与版本管理基础

接下来最值得投入的是“真实大模型可用性、Android 真机联调，以及 PC 客户端托盘 / 开机自启动等进一步产品化”。
